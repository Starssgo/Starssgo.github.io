调用函数指令栈空间变化

入栈规则

  • Cdecl

    参数入栈规则：从右到左
    还原栈：由调用者来修改栈

  • Stdcall

    参数入栈规则：从右到左。
    还原栈：由被调用函数来修改栈

  • Fastcall

    参数入栈：第一个跟第二个通过 ecx跟edx传递，剩余函数从右到左入栈
    还原栈：被调用者修改

Int A()  
{  
    B(x,y,z);  
}

调用前ebp与esp值

先将参数压入栈，按照从右到左原则入栈
Push 参数z
Push 参数y
Push 参数x
然后执行Call指令
Call指令将eip指令之后的指令压入栈，然后跳转到call之后的地址
push eip
eip->地址
此时栈空间如图
函数开始将ebp压入栈，将ebp设置为此时的esp，开辟局部变量栈空间
Push ebp
Mov ebp，esp

当函数结束后，首先取出令ebp为a栈ebp
Pop ebp
然后pop eip
此时程序跳转到eip对于的指令，即call下一个的指令
